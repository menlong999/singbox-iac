# Proposal

## Change ID

`smart-site-and-process-bundle-discovery`

## Why

Users should not need to know every related domain or helper process name for common developer tools and AI products.

The product already has useful proxifier bundles and some site aliases, but they need to grow into a first-class discovery layer.

## What Changes

- add a first-class site bundle registry for common services
- expand built-in process bundle coverage for common developer and AI tools
- let natural-language authoring resolve product names into site bundles and process bundles
- keep AI-assisted discovery optional and secondary to the built-in registry

## Out of Scope

- arbitrary web crawling
- silent AI-generated bundle edits without preview
- complete coverage of every site on the public internet
